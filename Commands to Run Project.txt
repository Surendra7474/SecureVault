=== LOCAL DEVELOPMENT ===

Backend:
  cd backend && mvn spring-boot:run
  (or) java -jar backend/target/securevault-backend-1.0.0.jar

Frontend:
  cd frontend && npm run dev


=== DEPLOYMENT: RENDER (Backend - H2 Database) ===

1. Create a Web Service on Render connected to your GitHub repo:
   https://github.com/Surendra7474/SecureVault.git

   Settings:
   - Name: securevault-api
   - Root Directory: backend
   - Runtime: Docker
   - (Or native Java 17: Build = ./mvnw clean package -DskipTests, Start = java -jar target/*.jar)

2. Add ENVIRONMENT VARIABLES on Render:

   SERVER_PORT=8080
   FRONTEND_URL=https://secure-vault-steel-theta.vercel.app
   JWT_SECRET=<your-base64-encoded-256-bit-key>
   VAULT_ENCRYPTION_KEY=<your-base64-encoded-256-bit-key>
   MAIL_USERNAME=<your-gmail>
   MAIL_PASSWORD=<your-gmail-app-password>

   (Generate secure keys: openssl rand -base64 32)


=== DEPLOYMENT: VERCEL (Frontend) ===

Already deployed at: https://secure-vault-steel-theta.vercel.app

1. Go to Vercel Dashboard → Settings → Environment Variables
2. Add:
   VITE_API_URL=https://<your-render-app>.onrender.com/api

3. Redeploy (Settings → Deployments → Redeploy latest)


=== PRODUCTION BUILD (Local) ===

Backend JAR:
  cd backend && mvn clean package -DskipTests

Frontend dist:
  cd frontend && npm run build
